## Шаг 1 — срез CSV (~10–15M строк) на Mac

```bash
ssh deproject-mac
mkdir -p ~/x5_slice
# распределение по месяцам (~2-3 мин на 4.5GB):
tail -n +2 ~/x5/purchases.csv | cut -d, -f3 | cut -c1-7 | sort | uniq -c
# выбрать НЕПРЕРЫВНОЕ окно последних месяцев с суммой 10-15M строк, затем:
head -1 ~/x5/purchases.csv > ~/x5_slice/purchases.csv
awk -F, 'NR>1 && $3 >= "<YYYY-MM-DD выбранной границы>"' ~/x5/purchases.csv >> ~/x5_slice/purchases.csv
cp ~/x5/clients.csv ~/x5/products.csv ~/x5_slice/
wc -l ~/x5_slice/purchases.csv   # убедиться: 10-15M
```

Loader не патчим: clients/products грузятся целиком (это dims, так и нужно).

## Шаг 2 — DDL raw vault в CH стенда

С Windows упаковать и доставить DDL:

```bash
cd /d/DE_project/warehouse/agentflow/dv2
tar -czf /tmp/dv2-ddl.tar.gz __init.sql raw_vault/hubs raw_vault/links raw_vault/satellites business_vault dbt/models/marts
scp /tmp/dv2-ddl.tar.gz deproject-mac:/tmp/ && ssh deproject-mac 'mkdir -p ~/dv2_ddl && tar -xzf /tmp/dv2-ddl.tar.gz -C ~/dv2_ddl'
```

На Mac применить (docker exec, HTTP-порт не нужен):

```bash
CH() { docker exec -i auto_bi_clickhouse clickhouse-client --user default --password "$CH_PASS" --multiquery; }
# __init.sql содержит DROP DATABASE IF EXISTS rv — на стенде rv нет, безопасно; dm не трогает.
cat ~/dv2_ddl/__init.sql | CH
for f in ~/dv2_ddl/raw_vault/hubs/*.sql ~/dv2_ddl/raw_vault/links/*.sql ~/dv2_ddl/raw_vault/satellites/*.sql; do cat "$f" | CH; done
echo "SELECT count() FROM system.tables WHERE database='rv'" | CH   # ожидаем 55
```

DDL писан под CH 24.x (README dv2) — со стендовым 24.8 совместим.

## Шаг 4 — BV (ВАЖНО: ловушка __1c__ vs __bitrix__)

Loader пишет header/pricing в `sat_order_header__1c__{branch}` / `sat_order_pricing__1c__{branch}`,
а ванильная вьюха `business_vault/bv_order_canonical.sql` читает `__bitrix__*` → header выйдет NULL.
Использовать **материализованную адаптацию** `~/x5_optionA/bv_order_canonical_mat.sql`
(проверить, какие сателлиты она читает; если bitrix — заменить на 1c). Скрипт
`~/x5_optionA/load_bv_order_canonical_mat.sh` написан под `kubectl exec` — заменить
функцию `ch()` на docker exec из шага 2 (рестарт-пляски с подом убрать; при OOM на
материализации — гнать по филиалам/месяцам частями).

Также применить `bv_customer_mdm__*.sql` (5 вьюх; читают `sat_customer_personal__1c__*` — ок).

## Шаг 5 — марты (DM «как есть»)

dbt на стенде не разворачиваем — 3 модели руками как CTAS в БД `marts`
(из `~/dv2_ddl/dbt/models/marts/*.sql`, заменив `{{ ref('...') }}`/`{{ source(...) }}`
на `rv.bv_order_canonical_mat` и `rv.bv_customer_mdm__*`):

```sql
CREATE DATABASE IF NOT EXISTS marts;
CREATE TABLE marts.customer_360    ENGINE=MergeTree ORDER BY (branch, customer_hk) AS <SQL модели>;
CREATE TABLE marts.branch_pnl      ENGINE=MergeTree ORDER BY (branch, month)        AS <SQL модели>;
CREATE TABLE marts.returns_velocity ENGINE=MergeTree ORDER BY (branch, channel, week) AS <SQL модели>;
```

Sanity: effective_tax_rate в branch_pnl ≈ 0.20 (msk/spb/ekb) / 0.05 (dxb) / 0.12 (ala).
Ничего не «улучшать» в мартах — их бедность и есть материал для gaps report.
